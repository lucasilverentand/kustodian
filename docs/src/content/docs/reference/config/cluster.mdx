Cluster configuration files define target Kubernetes environments with their templates, values, and node settings.

## Schema

```yaml
apiVersion: kustodian.io/v1
kind: Cluster
metadata:
  name: <cluster-name>
spec:
  defaults:
    flux_namespace: <string>
    oci_repository_name: <string>
    oci_registry_secret_name: <string>
    flux_reconciliation_interval: <duration>
    flux_reconciliation_timeout: <duration>
    flux_reconciliation_retry_interval: <duration>
  flux:
    source_ref:
      kind: <source-kind>
      name: <source-name>
      namespace: <source-namespace>
    interval: <duration>
    prune: <boolean>
    wait: <boolean>
    timeout: <duration>
    retry_interval: <duration>
    controllers:
      concurrent: <integer>
      requeue_dependency: <duration>
      max_retry_delay: <duration>
      feature_gates:
        <GateName>: <boolean>
      resources:
        limits:
          cpu: <string>
          memory: <string>
        requests:
          cpu: <string>
          memory: <string>
      tmpfs: <boolean>
      kustomize_controller:
        concurrent: <integer>
        requeue_dependency: <duration>
        max_retry_delay: <duration>
        no_remote_bases: <boolean>
        feature_gates:
          <GateName>: <boolean>
        resources: ...
        tmpfs: <boolean>
      helm_controller:
        concurrent: <integer>
        requeue_dependency: <duration>
        max_retry_delay: <duration>
        feature_gates: ...
        resources: ...
        tmpfs: <boolean>
      source_controller:
        concurrent: <integer>
        requeue_dependency: <duration>
        max_retry_delay: <duration>
        feature_gates: ...
        resources: ...
        tmpfs: <boolean>
  templates:
    - <template-name>
    - name: <template-name>
      values:
        <KEY>: <value>
      scheduling: ...
      kustomizations:
        <kustomization-name>:
          scheduling: ...
  values:
    <KEY>: <value>
  scheduling:
    node_selector:
      <label>: <value>
    affinity:
      node:
        required: [...]
        preferred: [...]
      pod: ...
      pod_anti: ...
    tolerations:
      - { key, operator, value, effect, toleration_seconds }
    topology_spread:
      - { max_skew, topology_key, when_unsatisfiable, label_selector, min_domains, match_label_keys }
    priority_class: <string>
    resources:
      requests: { cpu, memory }
      limits: { cpu, memory }
    containers:
      <container-name>:
        resources: { requests, limits }
    disabled: <boolean>
    workloads:
      <workload-name>:
        kind: <Deployment|StatefulSet|DaemonSet|Job|CronJob>
        node_selector: ...
        tolerations: ...
        resources: ...
        containers: ...
  node_defaults:
    label_prefix: <prefix>
    ssh:
      user: <username>
      port: <port>
      key_path: <path>
      known_hosts_path: <path>
  nodes:
    - <path>
```

## Fields

### apiVersion

### spec.flux

Flux reconciliation settings.

#### spec.flux.source_ref

| | |
|-|-|
| **Required** | Yes |
| **Type** | Object |

Flux source reference.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `kind` | String | Yes | Source type: `GitRepository`, `OCIRepository`, or `Bucket` |
| `name` | String | Yes | Source name |
| `namespace` | String | Yes | Source namespace |

#### spec.flux.interval

| | |
|-|-|
| **Required** | No |
| **Type** | String (duration) |
| **Default** | `10m` |

Reconciliation interval (e.g., `5m`, `1h`).

#### spec.flux.prune

| | |
|-|-|
| **Required** | No |
| **Type** | Boolean |
| **Default** | `false` |

Remove resources when deleted from source.

#### spec.flux.wait

| | |
|-|-|
| **Required** | No |
| **Type** | Boolean |
| **Default** | `true` |

Wait for resources to be ready.

#### spec.flux.timeout

| | |
|-|-|
| **Required** | No |
| **Type** | String (duration) |
| **Default** | `5m` |

Timeout for reconciliation.

#### spec.flux.retry_interval

| | |
|-|-|
| **Required** | No |
| **Type** | String (duration) |
| **Default** | `1m` |

Retry interval on failure.

### spec.defaults

Cluster-level defaults that override project defaults. All values are optional — fallback to project defaults, then schema defaults.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `flux_namespace` | String | `flux-system` | Flux system namespace |
| `oci_repository_name` | String | `kustodian-oci` | Flux OCIRepository resource name |
| `oci_registry_secret_name` | String | `kustodian-oci-registry` | OCI registry auth secret name |
| `flux_reconciliation_interval` | String (duration) | `10m` | Reconciliation interval |
| `flux_reconciliation_timeout` | String (duration) | `5m` | Reconciliation timeout |
| `flux_reconciliation_retry_interval` | String (duration) | `1m` | Retry interval for failed reconciliations |

### spec.flux.controllers

Tune Flux controller deployments. Settings can be applied globally or per-controller. Global settings are inherited by all controllers unless overridden.

| Field | Type | Description |
|-------|------|-------------|
| `concurrent` | Integer | Number of concurrent reconciliations |
| `requeue_dependency` | String (duration) | Requeue interval when waiting on dependencies |
| `max_retry_delay` | String (duration) | Maximum delay between retries |
| `feature_gates` | Object (map of string → boolean) | Flux feature gates passed as `--feature-gates` |
| `resources` | Object | Container resource requests/limits |
| `tmpfs` | Boolean | Mount `/tmp` as RAM-backed emptyDir (speeds up kustomize builds) |
| `kustomize_controller` | Object | Kustomize-controller-specific settings |
| `helm_controller` | Object | Helm-controller-specific settings |
| `source_controller` | Object | Source-controller-specific settings |

Per-controller objects accept the same fields as the global level (`concurrent`, `requeue_dependency`, `max_retry_delay`, `feature_gates`, `resources`, `tmpfs`). The `kustomize_controller` also accepts:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `no_remote_bases` | Boolean | `false` | Disable remote bases in Kustomize overlays |

**Merging behavior:**
- `feature_gates` — per-controller merges with global (per-controller wins on key conflicts)
- `resources` — per-controller replaces global entirely
- `tmpfs` — per-controller overrides global

### spec.templates

| | |
|-|-|
| **Required** | No |
| **Type** | Array |

Templates to deploy to this cluster.

Can be specified as:
- String: Template name
- Object: Template with value overrides

```yaml
templates:
  - nginx                    # Simple reference
  - name: app               # With overrides
    values:
      REPLICA_COUNT: "5"
```

### spec.scheduling

| | |
|-|-|
| **Required** | No |
| **Type** | Object |

Kubernetes scheduling defaults applied to every workload (`Deployment`, `StatefulSet`, `DaemonSet`, `Job`, `CronJob`) in every kustomization deployed by this cluster. Emitted as Kustomize strategic-merge `patches` on the generated Flux `Kustomization` resources.

The same block is available at three levels — each narrower level overrides the previous per field:

1. `spec.scheduling` — cluster-wide default
2. `spec.templates[].scheduling` — per template instance
3. `spec.templates[].kustomizations[<name>].scheduling` — per kustomization

Field merge semantics: scalars and lists **replace** at each lower level; `node_selector` and `resources.requests`/`resources.limits` **shallow-merge** (lower-level keys override). `disabled: true` at the kustomization level opts out of all inherited scheduling for that kustomization only.

| Field | Type | Description |
|-------|------|-------------|
| `node_selector` | Map[string,string] | nodeSelector applied to every workload |
| `affinity` | Object | Node/pod (anti-)affinity — see Kubernetes affinity docs |
| `tolerations` | Array | Pod tolerations |
| `topology_spread` | Array | Topology spread constraints |
| `priority_class` | String | `priorityClassName` applied to every workload |
| `resources` | Object | Requests/limits applied to the primary container (`name: main`) |
| `containers` | Map[string, { resources }] | Per-container resource overrides for multi-container pods |
| `disabled` | Boolean | (kustomization level) Skip emitting scheduling patches |
| `workloads` | Map[string, WorkloadScheduling] | Per-workload overrides keyed by `metadata.name` |

#### Resources and container targeting

`resources` at any level applies to a container named `main` by default. If your pod's primary container has a different name — or the pod has multiple containers — use `containers.<name>.resources` to target explicitly. Example:

```yaml
scheduling:
  containers:
    app:     { resources: { requests: { cpu: 500m } } }
    sidecar: { resources: { limits:   { memory: 64Mi } } }
```

#### Per-workload overrides

Workload-level entries are patched by name. Declare `kind` to emit a single targeted patch; omit it to emit one patch per workload kind (Kustomize no-ops mismatches):

```yaml
scheduling:
  priority_class: default
  workloads:
    database:
      kind: StatefulSet   # optional — when set, only one patch is emitted
      node_selector: { disk: nvme }
      tolerations: [{ key: dedicated, operator: Equal, value: db, effect: NoSchedule }]
      resources: { requests: { cpu: 2, memory: 4Gi } }
```

|-|-|

## Complete Example

```yaml
apiVersion: kustodian.io/v1
kind: Cluster
metadata:
  name: production
spec:
  flux:
    source_ref:
      kind: GitRepository
      name: flux-system
      namespace: flux-system
    interval: 10m
    prune: true
    wait: true
    timeout: 10m
  templates:
    - cert-manager
    - ingress-nginx
    - name: app
      values:
        REPLICA_COUNT: "5"
  values:
    ENVIRONMENT: production
    DOMAIN: app.example.com
    LOG_LEVEL: warn
  node_defaults:
    label_prefix: example.com
    ssh:
      user: deploy
      key_path: ~/.ssh/production_key
  nodes:
    - nodes/
```
